## **UC-03: 정서 일기 및 상담 자료 확인**

| 항목 | 내용 |
| --- | --- |
| **Use Case ID** | **UC-03** |
| **Use Case 명** | **정서 일기 회고 및 상담 자료 확인·공유** |
| **설명** | **사용자가 '일기'에 접속하면, 시스템이 아카이브(기록 계층)에서 순화된 정서 일기 목록을 날짜순으로 조회하고 필요 시 필터링하여 제공한다. 사용자는 선택한 일기의 상세(이야기체 텍스트 + 감정 수치 시각화)를 열람하고, 연결된 구조화 상담 문서를 확인·공유하거나 일기를 수정·삭제하며, 모든 변경 이력은 로그로 DB에 적재된다.** |
| **Primary Actor** | **일반 사용자 (청년/청소년)** |
| **Secondary Actor** | **심리 상담사 (상담용 문서 수신 시)** |
| Precondition | **UC-01이 완료되어 아카이브에 기록물(정서 일기 + 상담용 구조화 문서)이 1건 이상 저장된 상태** |
| Postcondition | **사용자가 자신의 감정 변화를 회고하고, 필요 시 상담사에게 구조화된 사전 자료가 전달되며, 일기의 수정·삭제 결과 및 로그가 DB에 반영된다.** |

### 기본 성공 시나리오 (Main Flow)

| 단계 | 계층 | 행위자 | 내용 |
| :-: | :-- | :--: | :-- |
| 1 | — | User | 홈화면에서 **'일기'에 접속**한다 |
| 2 | **기록 (아카이브)** | System | 아카이브의 **이전 기록 존재 여부**를 확인한다 (이전 기록 없음 시 2a 대안 시나리오 트리거) |
| 3 | **기록 (아카이브)** | System | **DB에서 일기 목록을 날짜순으로 조회**한다 `<<include>>` 일기 목록 조회 |
| 4 | — | System | 조회된 일기 목록을 표시한다 |
| 5 | — | User | **필터링 조건을 선택**한다 (미선택 시 5a 대안 시나리오 진행) `<<extend>>` 일기 목록 필터링 |
| 6 | **기록 (아카이브)** | System | 필터링 조건에 맞는 일기 목록을 재조회하여 표시한다 |
| 7 | — | User | **열람할 일기를 선택**한다 |
| 8 | — | System | 선택한 일기의 상세 내용(**이야기체 텍스트 + 감정 수치 시각화**)을 표시한다 |
| 9 | — | User | 일기 내용을 확인한다 (이후 동작 분기: 상담 문서 확인 / 공유 / 수정 / 삭제 / 나가기) |
| 10a | — | User | **"공유하기"** 를 선택한다 `<<extend>>` 상담 자료 공유 |
| 10b | **개인정보 보호** | System | **공유 방법을 표시**하고, 공유 전 마스킹 처리 여부를 재확인한다 |
| 10c | — | User · System | 사용자가 공유 방법을 선택하면, **상담 기관 공유용 포맷으로 문서를 내보낸다** |
| 11 | — | User | 열람·공유를 마치고 **홈화면** 또는 **일기 목록**으로 복귀한다 |

### 대안 시나리오 (Alternative Flow)

#### 2a. 저장된 기록 없음 (기록 계층)
| 단계 | 행위자 | 내용 |
| :-: | :--: | :-- |
| 2a.1 | System | 아카이브가 비어 있으면(이전 기록 없음) "아직 대화 기록이 없어요. 대화를 시작해보세요!" 안내를 표시한다 |
| 2a.2 | System | 사용자를 **UC-01 (음성 대화를 통한 감정 기록)** 로 유도한다 |

#### 5a. 필터링 조건 미선택
| 단계 | 행위자 | 내용 |
| :-: | :--: | :-- |
| 5a.1 | User · System | 필터링 조건을 선택하지 않으면, 전체 일기 목록 상태에서 기본 흐름 7단계(열람할 일기 선택)로 진행한다 |

#### 10b. 일기 수정 (기록 계층) `<<extend>>`
| 단계 | 행위자 | 내용 |
| :-: | :--: | :-- |
| 10b.1 | User | 일기 상세 확인 후 "수정"을 선택하여 **일기를 수정**한다 |
| 10b.2 | User | "저장"을 선택한다 (**"수정 취소"** 선택 시 변경 없이 일기 상세 화면으로 복귀) |
| 10b.3 | System | **수정 내용을 DB에 반영**한다 |
| 10b.4 | System | **수정 로그를 DB에 저장**한다 |
| 10b.5 | System | 수정 완료를 안내하고 일기 상세 화면으로 복귀한다 |

#### 10c. 일기 삭제 (기록 계층) `<<extend>>`
| 단계 | 행위자 | 내용 |
| :-: | :--: | :-- |
| 10c.1 | System | 일기 상세 확인 후 "삭제"를 선택하면 **"삭제하면 복구할 수 없습니다"** 경고를 표시한다 |
| 10c.2 | User | "삭제"를 선택하여 진행한다 (**"취소"** 선택 시 일기 상세 화면으로 복귀) |
| 10c.3 | User | 삭제 범위를 선택한다 — (a) **일기만 삭제** / (b) **연결 상담 문서 동반 삭제** |
| 10c.4 | System | (b 선택 시) 연결된 **상담 문서를 먼저 삭제**한다 |
| 10c.5 | System | **일기를 삭제**한다 |
| 10c.6 | System | **삭제 로그를 DB에 저장**한다 |
| 10c.7 | System | 삭제 완료를 안내한다 |

### 관련 유스케이스

- `<<include>>` 일기 목록 조회 (기록 계층 - 아카이브)
- `<<extend>>` 일기 목록 필터링 (목록 표시 단계)
- `<<extend>>` 상담 문서 확인 (일기 상세 열람 단계)
- `<<extend>>` 상담 자료 공유 (Secondary Actor: 심리 상담사 / 외부 전문 상담 기관)
- `<<extend>>` 일기 수정 (기록 계층 - 수정 로그 적재)
- `<<extend>>` 일기 삭제 (기록 계층 - 삭제 로그 적재)
- **UC-01** 음성 대화를 통한 감정 기록 (Precondition - 기록물 생성 / 기록 없음 시 유도)
