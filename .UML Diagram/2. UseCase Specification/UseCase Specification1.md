## **UC-01 대화 기반 내면 탐색 및 기록**

| 항목 | 내용 |
| --- | --- |
| **Use Case ID** | **UC-01** |
| **Use Case 명** | **음성 대화를 통한 감정 기록** |
| **설명** | **사용자가 AI와 음성 대화를 통해 자신의 내면을 탐색하면, 시스템이 4-Layer 엔진(사전처리·기록·안전 프로토콜·상호작용)을 통해 멀티모달 감정 분석을 수행하고, 대화 종료 시 순화된 정서 일기와 상담용 구조화 문서를 자동 생성하여 DB에 저장** |
| **Primary Actor** | **일반 사용자 (청년/청소년)**  |
| **Secondary Actor** | **외부 전문 상담 기관** |
| Precondition | **사용자가 로그인되어 있고, 음성 녹음 권한에 동의한 상태** |
| Postcondition | **정서 일기와 상담용 구조화 문서가 DB에 저장되고, 사용자에게 하루 감정 요약이 제공** |

### 기본 성공 시나리오 (Main Flow)

| 단계 | 계층 | 행위자 | 내용 |
| :-: | :-- | :--: | :-- |
| 1 | — | User | 홈화면에서 "대화하기"를 선택한다 |
| 2 | — | System | 마이크를 활성화하고 대화 세션을 시작한다 |
| 3 | — | User | 오늘의 감정/상황을 음성으로 말한다 |
| 4 | **사전처리** | System | **OpenAI API (STT)** 로 음성을 실시간 텍스트로 변환한다 |
| 5 | **사전처리** | System | **KoELECTRA NER 모델** 로 개인정보(이름·장소·날짜 등)를 자동 마스킹한다 `<<include>>` 음성→텍스트 변환 및 개인정보 마스킹 |
| 6 | **기록 (감정 분석)** | System | **Fork**: 두 분석을 병렬 수행한다<br>① **OpenAI API**: 텍스트 문맥 감정 분석 (7가지 감정 + 강도 수치화)<br>② **Hume AI Prosody**: 음성 떨림·피치·에너지 분석 |
| 7 | **기록 (감정 분석)** | System | **Join**: 두 분석 결과를 캘리브레이션하여 종합 감정 지표를 생성한다 `<<include>>` 멀티모달 감정 분석 (텍스트 + 음성 교차 검증) |
| 8 | **안전 프로토콜** | System | 실시간 위험도를 판단한다 (저/중위험은 정상 흐름 진행, 고위험은 8a 대안 시나리오 트리거) `<<extend>>` 위기 감지 및 안전망 연결 (UC-02) |
| 9 | **상호작용** | System | **OpenAI API** 로 감정 강도와 대화 맥락 기반 꼬리물기 질문을 동적 생성하고 **ElevenLabs API (TTS)** 로 음성 응답한다 `<<include>>` 꼬리물기 질문 생성 |
| 10 | — | User | AI 응답을 듣고 답변을 이어간다 |
| 11 | — | User · System | 3~10단계를 대화 루프로 반복한다 (5단계 구조: 상황 파악 → 감정 탐색 → 생각 탐색 → 패턴 연결 → 마무리) |
| 12 | — | User | "대화 끝내기"를 선택한다 |
| 13 | **기록 (저장)** | System | **Fork**: 두 기록물을 병렬 생성한다<br>① **정서 일기 생성**: 순화 알고리즘으로 대화를 가공하여 이야기체 정서 일기 생성<br>② **상담용 문서 생성**: 감정 프로필·인지왜곡 패턴·비언어 지표 포함한 구조화 자료 생성 |
| 14 | **기록 (저장)** | System | **Join**: 두 기록물을 **DB에 저장**한다 `<<include>>` 이중 기록물 자동 생성 (정서 일기 + 상담 자료) |
| 15 | — | User | 하루 요약과 함께 두 기록물을 확인한다 |

### 대안 시나리오 (Alternative Flow)

#### 4a. 음성 인식 실패 (사전처리 계층)
| 단계 | 행위자 | 내용 |
| :-: | :--: | :-- |
| 4a.1 | System | OpenAI API (STT)가 음성을 인식하지 못하면 재시도 안내를 표시한다 |
| 4a.2 | System | 반복 실패 시 텍스트 입력 모드로 전환한다 |
| 4a.3 | User · System | 기본 흐름 5단계로 복귀한다 |

#### 8a. 실시간 고위험 키워드 감지 (안전 프로토콜 계층) `<<extend>>`
| 단계 | 행위자 | 내용 |
| :-: | :--: | :-- |
| 8a.1 | System | 자살·자해 관련 고위험 키워드 또는 임계치 초과 감정 지표 감지 시 대화 루프를 **즉시 중단**한다 |
| 8a.2 | System | **UC-02 (종합 위험도 평가 및 안전망 연결)** 를 즉시 실행한다 |
| 8a.3 | System | UC-02 종료 후, 부분 대화 내용이라도 정서 일기와 상담용 문서로 저장된다 (기본 흐름 13~14단계 수행) |

### 관련 유스케이스

- `<<include>>` 음성→텍스트 변환 및 개인정보 마스킹 (사전처리 계층)
- `<<include>>` 멀티모달 감정 분석 (기록 계층 - 분석)
- `<<include>>` 꼬리물기 질문 생성 (상호작용 계층)
- `<<include>>` 이중 기록물 자동 생성 (기록 계층 - 저장)
- `<<extend>>` **UC-02** 종합 위험도 평가 및 안전망 연결 (안전 프로토콜 - 실시간 고위험 감지 시)
